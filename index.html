REST را یاد بگیر

ساختن یک سرویس وب RESTful همانند همه مهارت های برنامه نویسی به دانش و دوق نیاز دارد. با گسترش اینترنت استفاده از API های REST هم بیشتر و بشتر شده و به مرور با best practice ها در هم آمیخته شده است. در توسعه API های RESTful شما مجبور نیستید که از هیچ استاندار و پروتکل خاصی تبعیت کنید (البته بجز HTTP) بنابراین برای جلوگیری از نوشتن کدهای غیراستاندارد که موجب می شود که کلاینت ها برای توسعه به زحمت بیوفتند و یا گاهی با مشکل روبرو شوند بهتر است که یک سری استاندارد تعریف شود تا توسعه دهندگان سرویس های RESTful از آن تبعیت کنند. در صورت استفاده از این ادبیات استاندارد در توسعه، کلاینت ها هم مشکلی با اتصال به این API ها نخواهند داشت و البته این استانداردها مزایای دیگری هم دارند از جمله افزایش توسعه پذیری نرم افزار و موارد دیگر که در ادامه می بینید. به این استانداردها به اصطلاح best practice می گویند، اینها در واقع تجربیات موفق جامعه نرم افزاری می باشند که همگی سر استفاده از آنها به توافق رسیده ایم و از مزایای استفاده از آنها بهره مند می شویم. 

 

در حال حاضر منابع آموزشی زیادی برای یاگیری REST API در اختیار برنامه نویسان تنها وجود ندارد. وبسایت RestApiTutorial.com همواره منابع مرتبط با REST API و best practice ها را جهت خودآموزی در دسترس برنامه نویسان قرار می دهد. ما به هر دو بخش علمی و هنری آن می پردازیم.

--تاد فردریچ، آموزگار REST API

 

من هم سعی می کنم تا جایی که بتونم آموزش های این وبسایت را به فارسی ترجم کنم و در RestApiTutorial.ir قرار بدم. اگر شخصی علاقه مند باشه و به آدرس گیتهاب این وبسایت بره و تغییری بده یا ترجمه ای اضافه کنه خیلی خوشحال می شم.

حالا بریم به صفحه بعدی آموزش که مفاهیم و محدودیت های معماری RESTful رو ببنیم.
